# Data usage

The **Data Usage** documentation outlines Claude Code’s data handling policies, transparency, and workflows for both consumer and commercial users. It details how user data (prompts, outputs, and interactions) is processed, retained, and optionally used for model training under specific conditions. **Consumer users** (Free/Pro/Max plans) can opt into sharing data for model improvements, while **commercial users** (Team/Enterprise/API) retain default privacy unless explicitly opting into programs like the **Development Partner Program**. Key sections cover:
- **Data policies**: Differentiated rules for consumer vs. commercial data usage, including restrictions on training with third-party API providers (e.g., Bedrock, Vertex).
- **Feedback mechanisms**: Transcripts from `/bug` reports are retained for 5 years, while session quality surveys (numeric ratings only) are anonymized and do not impact training preferences.
- **Data flow**: Clarifies distinctions between **local execution** (on-device processing) and **cloud execution** (API-dependent data handling), including dependencies like telemetry services.
- **Retention/access**: Data retention periods, user access controls, and opt-out options (e.g., disabling surveys via `CLAUDE_CODE_DISABLE_FEED
