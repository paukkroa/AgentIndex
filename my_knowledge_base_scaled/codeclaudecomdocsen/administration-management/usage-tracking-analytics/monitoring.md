# Monitoring

The **Monitoring** documentation for Claude Code provides a comprehensive guide to tracking system usage, costs, and tool activity via **OpenTelemetry (OTel)**. It covers **metrics (time-series data)** and **events/logs** exported through configurable backends (e.g., OTLP, Prometheus, or console). Key sections include **administrator configuration** (e.g., dynamic headers, multi-team support), **metrics cardinality control**, and **event correlation attributes** (e.g., user prompts, API errors). Advanced topics like **alerting, segmentation, and ROI measurement** are detailed, alongside backend considerations for scalability and security. Example configurations and counters (e.g., lines of code, token usage) are provided for granular monitoring. The guide also addresses **Amazon Bedrock integration** and **privacy/security** best practices.
