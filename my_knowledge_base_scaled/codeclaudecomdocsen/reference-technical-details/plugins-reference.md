# Plugins reference

The **Plugins reference** provides an exhaustive technical breakdown of the Claude Code plugin architecture, covering core components like **skills** (custom `/name` shortcuts defined via `SKILL.md` files in `skills/` or `commands/` directories), **agents** (task-specific subagents in `agents/` with markdown-based definitions), **hooks** (event-driven extensions), **MCP/LSP servers** (custom server integrations), and **plugin manifest schemas** (structured metadata, paths, and environment variables). It details **CLI commands** (`plugin install`, `plugin update`, etc.), **directory structures**, **caching mechanisms**, **security restrictions** (path traversal limitations), and **debugging tools** (e.g., troubleshooting hooks or MCP servers). The reference also outlines **versioning**, **distribution**, and **common pitfalls**, with cross-references to plugin creation, discovery, and marketplace distribution.
