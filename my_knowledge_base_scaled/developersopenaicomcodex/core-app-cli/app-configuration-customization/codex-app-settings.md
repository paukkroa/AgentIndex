# Codex app settings

The **Codex app settings** panel allows users to customize the app’s behavior, file handling, and integrations via a modular interface. Under **General**, users can configure file opening behavior (e.g., default editor) and control command output visibility, including requiring **Cmd+Enter** for multiline prompts or disabling sleep during active threads. **Appearance** settings let users switch themes, adjust window transparency, and modify UI/terminal fonts globally. **Notifications** enable granular control over completion alerts and permission prompts. **Agent configuration** ties to IDE/CLI settings, with in-app tweaks or manual edits to `config.toml` for advanced options (referencing [security](https://developers.openai.com/codex/security) and [config guides](https://developers.openai.com/codex/config-basic)). **Git** settings standardize branch naming, commit prompts, and force-push policies. **Integrations & MCP** facilitate external tool connections via OAuth or custom servers, syncing with CLI/IDE via `config.toml` (see [MCP docs](https://developers.openai.com/codex/mcp)). **Personalization** offers three default agent personalities (**Friendly**, **Pragmatic**, **None**) or custom
