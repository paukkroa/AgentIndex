# Introduction¶

The **Python Standard Library** encompasses core language components—such as built-in data types (e.g., numbers, lists), functions, and exceptions—that extend Python’s core semantics without requiring explicit imports. While some elements (e.g., operators, basic literals) are defined by the language itself, others (e.g., advanced functions like `print()` or `Exception` classes) are documented here but not strictly essential to core functionality. The bulk of the library consists of **modules**—ranging from Python-written or C-embedded implementations—categorized by purpose: Python-specific utilities (e.g., stack traces), OS-specific hardware/OS access (e.g., file I/O), or domain-specific applications (e.g., web, math). Modules vary in availability: some are universal across Python versions/platforms, while others depend on system support (e.g., Linux kernel/glibc versions) or compilation options. The manual’s **"inside-out" structure** prioritizes built-in functions/data types first, followed by modular chapters, enabling progressive learning. Availability notes clarify cross-platform compatibility, including Unix-based systems (macOS/iOS/Android) and WebAssembly constraints.
