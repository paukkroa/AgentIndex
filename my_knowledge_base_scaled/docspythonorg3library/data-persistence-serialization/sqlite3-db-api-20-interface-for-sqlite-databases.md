# sqlite3— DB-API 2.0 interface for SQLite databases¶

The `sqlite3` module provides a **DB-API 2.0-compliant** interface for interacting with **SQLite databases** in Python, leveraging the lightweight, serverless SQLite C library. It enables SQL-based data persistence without requiring a separate server process, making it ideal for embedded applications or prototyping before migrating to larger systems like PostgreSQL or Oracle. Key features include **transaction support**, **flexible typing** (optional data type declarations), and **cursor-based query execution**. The module adheres to **PEP 249**, offering standard database operations like `connect()`, `cursor()`, and `execute()` for CRUD tasks. Optional dependencies include the SQLite library, which must be installed separately. The documentation is structured into **tutorials** (e.g., Monty Python movie database example), **reference guides** (API classes/functions), **how-to guides**, and **transaction explanations**, catering to both beginners and advanced users.
