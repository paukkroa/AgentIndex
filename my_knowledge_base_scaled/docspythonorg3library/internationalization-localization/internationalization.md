# Internationalization¶

This page outlines Python’s **Internationalization (i18n)** modules, which enable language and locale-independent software development. It covers core modules like **`gettext`** (for multilingual support via translation catalogs, including GNU API, class-based APIs like `GNUTranslations` and `NullTranslations`, and dynamic language switching) and **`locale`** (for locale-specific formatting, names, and message catalog access). Key features include localization techniques (module/application-level), deferred translations, and integration notes for extension writers. The documentation emphasizes practical implementation, caveats, and system-specific considerations (e.g., Solaris support).
